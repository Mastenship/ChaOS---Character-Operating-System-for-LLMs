=== SECTION: CANONICAL_TOKEN_DICTIONARY ===

[KEEP: Purpose & Role]
This module is the single source of truth for all canonical bracket-path tokens used throughout the character system. It defines:
- Canonical token style (machine-stable UPPER_SNAKE)
- Canonical module roots
- Canonical group and field tokens (high-level)
- Allowed alias translations for legacy or human-readable tokens
- Normalization rules used by STATE_GOVERNANCE_AND_COMMIT

[KEEP: Relationship to Other Modules]
Informs: ALL modules and COT templates that read or write bracket paths.
Is Informed By: Nothing. This is a system constitution.

=== SECTION_BREAK ===

=== SECTION: CANONICAL_STYLE_RULES ===

[[LLM_INSTRUCTION]]: Canonical bracket-path tokens MUST be machine-stable.

TOKEN_STYLE:
- UPPER_SNAKE only
- No spaces
- No hyphens
- No mixed case
- IDs remain uppercase with underscores (ENT_01, SKL_03, ALT_01, MEM_42_01, PRD_01, GOAL_01)

CANONICAL_PATH_FORMAT:
- [MODULE_ROOT][ID_OR_GROUP][FIELD]
- Nested groups extend with additional brackets:
  - [MODULE_ROOT][ID_OR_GROUP][SUBGROUP][FIELD]

PROHIBITED:
- Any token containing spaces
- Any token containing ":" as a pseudo-path separator
- Any token using Title Case inside brackets
